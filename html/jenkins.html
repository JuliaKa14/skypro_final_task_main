Jenkins

Основные понятия Jenkins

  • Job/Pipeline — управление задачами или конвейерами
  • Build — процесс сборки
  • Workspace — каталог для сборки проектов
  • Node/Agent — упрпавление узлами
  • Plugin — это пакеты (.hpi-файлы), которые расширяют возможности Jenkinsи
  • Parameterized Build — функция, позволяющая запускать задание с передачей ему параметров

Типы проектов в Jenkins

  • Freestyle project — классический и универсальный тип задачи, который настраивается вручную через графический интерфейс (GUI)
  • Pipeline — набор плагинов, позволяющий описать весь процесс сборки, тестирования и развертывания приложения как код, который хранится в файле
  • Multibranch Pipeline — тип проекта, который автоматически обнаруживает, управляет и запускает отдельные конвейеры для каждой ветви репозитория
  • GitHub Organization — гибридное название, которое описывает интеграцию Jenkins (инструмент автоматизации CI/CD) с GitHub

Основные этапы CI/CD

  • Source Code Management — интеграция с системами контроля версий, такими как Git, для автоматизации процессов CI/CD
  • Build Triggers — настройки, которые автоматически запускают выполнение сборки проекта в ответ на определённые события
  • Build Environment — переменные окружения, параметры и другие настройки, необходимые для автоматизированного процесса сборки, тестирования и развертывания программного обеспечения
  • Build Steps — отдельные, последовательные действия, из которых состоит задача (job) или конвейер (pipeline)
  • Post-build Actions — действия, которые выполняются после завершения основного процесса сборки (jobs), этапа (stage) или пайплайна
